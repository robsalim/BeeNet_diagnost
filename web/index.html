📖 Инструкция пользователя

🎯 Назначение

Мониторинг серверов сбора данных "Береза" и "Малые точки", проверка доступности модемов, преобразователей MOXA Nport и просмотр информации о связи прибора учета с модемом или преобразователем MOXA.

📊 Вкладка "Мониторинг серверов"

  • IServer.exe - статус процесса сбора данных (зеленый - запущен, красный - остановлен)
  • Web_Bee.Api - статус API сервера, PID процесса и пользователь
  • Синхронизация времени - разница между системным временем и NTP сервером
  • База данных - статус SQL соединения и XML сообщений
  • Задержки данных - точки сбора с задержкой более 4 часов
  • Перезапуск/Остановка - управление IServer (доступно если API сервер запущен от администратора)

🔌 Вкладка "Тест модемов"

  • Тест TCP - проверка доступности порта модема (таймаут 10 сек)
  • Тест PING - проверка доступности по ICMP (5 попытки, таймаут 6 сек)
  • Кликабельный IP - при нажатии на IP запускается быстрый PING

📋 Вкладка "Приборы учета"

  • Список всех приборов учета
  • Поиск - по названию, IP, местоположению, номеру прибора, типу
  • Кликабельный IP - при нажатии на IP запускается быстрый PING
  • Кликабельный номер счетчика - при нажатии на `Номер счетчика` запускается тест `Echo` (доступно в режиме модема `Сервер` )

⚠️ Важная информация

Кнопка "Остановить": полностью останавливает процесс IServer.exe. После остановки сбор данных прекращается! Для запуска используйте "Перезапустить".
Кнопка "Перезапустить": останавливает и запускает IServer.exe заново. Используется при зависаниях или после изменения конфигурации.
Автообновление: данные обновляются автоматически каждые 30 секунд. Также можно обновить вручную кнопкой "🔄 Обновить" в правом нижнем углу.

🖥️ Требования к правам

  • Для перезапуска/остановки IServer необходимы права администратора
  • Для запуска API сервера (Web_Bee.Api) необходимы права администратора
  • Доступ к сети 172.16.56.x для тестирования модемов

📞 Контакты

При возникновении проблем обращаться к администратору системы.
