# BeeDotNet Monitor v2.0

Мониторинг двух независимых серверов сбора данных и проверка доступности модемов.

## Функции

### 🔍 Мониторинг серверов
- Проверка статуса IServer.exe на каждом сервере
- Мониторинг Web_Bee.Api (PID, пользователь)
- Синхронизация времени (сравнение с сервером)
- Мониторинг базы данных (SQL, XML)
- Проверка задержек данных
- Список точек с задержкой

### 🔌 Тест модемов
- TCP тест (проверка порта)
- PING тест (ICMP, 5 попыток)
- Группировка модемов по портам/группам
- Отображение задержки

### ⚡ Управление
- Перезапуск IServer для каждого сервера отдельно
- Кнопка перезапуска доступна только администраторам (пользователь с `-a-`)
- Планировщик автоматической перезагрузки (если есть задержки данных)

### ⏰ Планировщик
- Настраиваемое время перезагрузок (например, 02:50, 06:50)
- Перезагружается только сервер с задержками данных
- Логирование всех действий

### 🌐 Веб-интерфейс
- Две вкладки: "Мониторинг серверов" и "Тест модемов"
- Автообновление каждые 30 секунд
- Визуальные индикаторы статуса (зеленый/красный)

## Конфигурация

### Файл `config/servers.json`
```json
{
  "servers": [
    {
      "name": "Сервер Береза (10.96.30.62)",
      "address": "http://10.96.30.62:9200",
      "port": 9200
    },
    {
      "name": "Сервер Малые точки (10.98.30.36)",
      "address": "http://10.98.30.36:9200",
      "port": 9200
    }
  ],
  "web_port": ":8090",
  "check_interval_seconds": 30,
  "time_diff_threshold_seconds": 3,
  "scheduler_enabled": true,
  "restart_times": ["02:50", "06:50"]
}
```

### Файл `config/modems.json`
```json
{
  "modems": [
    { "ip": "172.16.56.160", "port": 5555, "name": "мотив", "group": "будка" },
    { "ip": "172.16.56.161", "port": 5555, "name": "РП5", "group": "будка" }
  ],
  "timeout_seconds": 10
}
```

## API Endpoints
Метод	Endpoint				Описание

GET		/api/full-status		Полный статус всех серверов

GET		/api/all-servers		Упрощенный статус серверов

GET		/api/db-health			Статус БД для сервера

GET		/api/data-delays		Задержки данных для сервера

GET		/api/privileges			Информация о пользователе

POST	/api/restart			Перезапуск IServer

POST	/api/modems-tcp-test	TCP тест модемов

POST	/api/modems-ping-test	PING тест модемов


## Логирование
Логи выводятся в консоль:
	Статус мониторинга
	Результаты проверок
	Действия планировщика
	Ошибки подключения
